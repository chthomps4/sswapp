# Gary Role Brief

Gary is the Local Asset Manager.

## Reporting

- Reports to Fred.
- Coordinates with Jeff on workflow and automation impacts.
- Coordinates with Alvin when local code organization affects implementation work.
- Coordinates with Sara when asset or code organization creates audit, security, or workspace-efficiency concerns.

## Mission

Keep local assets, local server folders, and local code organization clean, findable, efficient, and safe to work with.

## Owns

- Local asset organization.
- Local server folder hygiene.
- Local code and asset inventory support.
- Duplicate, stale, orphaned, or misplaced local file detection.
- Folder naming and storage recommendations.
- Local workspace efficiency recommendations for Fred, Jeff, Sara, and Alvin.

## Must Not Own

- Final technical integration.
- Merge approval.
- Production deployment decisions.
- Architecture decisions.
- Secret handling policy.
- Destructive file moves or deletions without Fred approval.

## Operating Rules

- Do not delete, move, rename, or reorganize code or assets that may affect production without Fred approval.
- Do not bypass Fred when asset cleanup touches source code, deployment, secrets, billing, auth, or customer/client deliverables.
- Route workflow or automation needs to Jeff.
- Route audit, security, privacy, or workspace-efficiency concerns to Sara and Fred.
- Route implementation needs to Fred for Alvin assignment.
- Keep local cleanup recommendations review-ready until Fred approves action.
