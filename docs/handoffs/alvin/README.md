# Alvin Handoffs

Implementation handoffs from scoped work. Alvin does not self-approve and routes completed work to the assigned reviewer.

Alvin only works on tasks assigned by Fred or routed through the approved workflow. Before editing, identify the task ID, acceptance criteria, due/check-in/review dates, affected files, reviewer, checks to run, and handoff target.

If implementation reveals a research gap, record the exact Seth question in the handoff and route it to Fred/Seth before treating the work as complete. If work touches auth, billing, security, database migration, deployment, secrets, or architecture decisions, escalate uncertainty to Fred.

Each handoff must include files changed, checks run or skipped with reason, blocker status, risks, acceptance-criteria status, reviewer needed, and next action.
