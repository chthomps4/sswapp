# Handoff Protocol

## Routing Rules

- Final technical integration goes to Fred.
- CEO/Owner approval goes to Fred first, then Fred reports to CEO/Owner.
- Automation, scripts, handoff mechanics, GitHub Actions, recurring checks, or process reliability go to Jeff.
- Audit, quality, security, test, deployment risk, billing risk, deadline risk, CEO bottleneck risk, or release readiness go to Sara.
- Research, market analysis, competitor analysis, tool/vendor evaluation, prototype discovery, technical feasibility, or product opportunity discovery go to Seth.
- Workflow design, diagrams, step-by-step process, or handoff format go to Ed through Jeff.
- Scoped implementation goes to Alvin.
- Local asset management, local server folder organization, local code/file hygiene, duplicate/stale local file detection, or workspace asset efficiency goes to Gary through Fred.

## R&D Routing

- Seth sends implementation recommendations to Fred for assignment to Alvin.
- Seth sends automation/workflow recommendations to Jeff.
- Seth sends workflow-design recommendations to Jeff for Ed.
- Seth sends audit/risk concerns to Sara and Fred.
- Seth sends tool/subscription/budget needs to Fred as `CEO ACTION REQUIRED`.
- Seth research is not production approval.

## Deadline Routing

- Overdue work goes to Fred.
- At-risk launch, revenue, customer, security, or deployment work goes to Fred immediately.
- CEO-action-needed work goes to Fred immediately.
- Missing research blockers go to Seth and Fred.
- Workflow/process blockers go to Jeff.
- Workflow design blockers go to Jeff for Ed.
- Audit blockers go to Sara.
- Scoped implementation blockers go to Alvin; unclear implementation blockers go to Fred.
- Local asset or local server organization blockers go to Gary through Fred.

## Command Intake Routing

- Slack `#command` is intake only.
- Linear is the execution board.
- Repo docs and handoffs are durable memory.
- Codex may draft code, docs, tests, summaries, and tracker notes, but all outputs remain review-ready until a human approves them.
- Fred reviews task routing before implementation, merge, deployment, publishing, or business-impacting action.
- Jeff owns the intake/delegation workflow and routes Ed workflow design through Jeff.
- Business-model, pricing, launch, vendor, paid-tool, or public-claim decisions route from Fred to the CEO/Owner.

## Required Handoff

Every meaningful task ends with a handoff in the relevant role folder and an entry in `docs/chat-handoff-log.md`.
